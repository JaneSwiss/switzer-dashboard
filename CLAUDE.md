## Brand Voice

**Tone:** practical, natural, human, benefit-led, grounded, slightly direct.
Never robotic, vague, fluffy, overly motivational, or generic AI-sounding.

**Writing rules:**
- Connect design to business outcomes always
- Make benefits obvious: look professional, save time, attract clients,
  build trust, drive traffic, make sales, strengthen branding
- Be specific — avoid generic praise
- Treat the audience as smart but busy — make the message clear quickly
- Short sentences. No padding. No filler.
- Every sentence earns its place.

**Hooks that work:** short, direct, call out a business owner's frustration,
realization, or ambition. E.g. "Your Instagram isn't the problem. Your brand is."

**Never use:** "clarity", "strategy", "content system" without a concrete benefit
attached. Never use repetitive structures across captions or pin titles.

## Output Standards

Every piece of content produced must:
- Reflect brand voice (practical, human, benefit-led, never robotic)
- Use brand colours where visual output is involved
- Connect design or marketing advice to a business outcome
- Be written for a female small business owner audience
- Never mention AI, automation, or that it was generated

Pinterest copy must:
- Include primary keyword in title (naturally, not stuffed)
- Description: 150-300 characters, keyword-rich, benefit-led
- No spammy repetition across pins

Email copy must:
- Subject line: under 50 characters, curiosity-driven or benefit-led
- Preview text: adds to subject line, never repeats it
- Body: short paragraphs, no walls of text, one clear CTA

Blog posts must:
- Target a specific commercial-intent keyword
- Include a natural soft-sell CTA to a relevant product
- Be 800-1500 words unless brief is specified otherwise
- Be written for Google and Pinterest traffic, not just social

## Skills Directory

Python automation scripts in `skills/`. Run from project root or their own directory.
All require: `pip3 install anthropic python-dotenv pandas beautifulsoup4 requests reportlab`

### `skills/etsy-trend-analysis/`
**What it does:** Analyses Everbee CSV exports to surface keyword opportunities, dead listings, product gaps, and competitor insights. Generates a full markdown report + branded PDF.

**Run:** `python3 skills/etsy-trend-analysis/main.py`

**Inputs:**
- Drop Everbee keyword CSVs and/or listing CSVs into `data/everbee-etsy/`
- Keyword CSVs: columns `Keyword, Volume, Competition, Keyword Score`
- Listing CSVs: columns `Product Name, Est. Sales, Est. Revenue, Total Views, Total Favorites, Conversion Rate, Visibility Score, Shop Name`
- If all rows in a listing CSV have `shop_name = switzertemplates` → treated as own-shop data
- Otherwise → treated as competitor/market data

**Outputs:**
- `outputs/etsy-sales/etsy-trend-report-YYYY-MM-DD.md` — full report
- `outputs/etsy-sales/etsy-trend-report-YYYY-MM-DD.pdf` — branded PDF with clickable links

**Key logic:**
- Product gap detection uses exact phrase matching (not word-by-word) against own tags and titles
- Competition column = raw listing count, not a score — uses Keyword Score directly for ranking
- Dead listings = >5000 views, zero sales
- Action plan always targets 30 sales/day goal

---

### `skills/keyword-preprocessor/`
**What it does:** Cleans and scores Everbee keyword CSV exports before feeding them to other skills. Removes duplicates, flags anomalies, filters by min score, marks valid Etsy tags (≤20 chars).

**Run:** `python3 skills/keyword-preprocessor/main.py <csv_path> [--min-score 40] [--output markdown|csv|both]`

**Inputs:** Any Everbee keyword CSV export

**Outputs:**
- `<input>.processed.md` — three sections: HIGH OPPORTUNITY (100+), MID OPPORTUNITY (40–99), VALID ETSY TAGS (≤20 chars)
- `<input>.processed.csv` — cleaned DataFrame with added `chars` and `valid_etsy_tag` columns

**Used by:** `report-generator` (loads `.processed.md` if available, else runs preprocessor on raw CSV), `etsy-listing-optimizer` (same fallback logic)

---

### `skills/etsy-competitor-research/`
**What it does:** Scrapes Etsy listing pages to extract title, price, tags, sales, rating, and description preview. Outputs a markdown report with tag overlap analysis across all scraped listings.

**Run:**
- `python3 skills/etsy-competitor-research/main.py listings <url> [url...]`
- `python3 skills/etsy-competitor-research/main.py search "<keyword>" --top N`

**Inputs:** Etsy listing URLs or a search keyword

**Outputs:** Markdown report saved to `data/competitors/<slug>.md`

**Notes:** 1.2s polite delay between requests. Skips own shop URLs. No API key needed — uses HTML scraping + JSON-LD extraction.

---

### `skills/report-generator/`
**What it does:** Combines Everbee keyword data + competitor markdown files + shop config, sends to Claude, and saves a bi-weekly Etsy Growth Report.

**Run:** `python3 skills/report-generator/main.py [--everbee path/to/export.csv]`

**Inputs:**
- Everbee CSV in `data/everbee-etsy/` (uses most recent if not specified)
- Competitor reports in `data/competitors/*.md` (up to 3 most recent)
- `context/shop-config.json` if exists, otherwise falls back to `DEFAULT_SHOP_CONFIG` hardcoded in the script

**Outputs:**
- `reports/etsy-growth-report-YYYY-MM-DD.md`
- `reports/latest-etsy-report.md` (overwritten each run — used by scheduler/dashboard)

**Report structure:** Shop Snapshot → Priority Task List (1–10 by revenue impact) → Keyword Opportunities → Competitor Intelligence → Dead Listings to Fix or Cut

---

### `skills/etsy-listing-optimizer/`
**What it does:** Takes a single listing's title, tags, and description, combines with Everbee keyword data, and asks Claude to rewrite tags, description opening, and title for better SEO and conversion.

**Run:**
- `python3 skills/etsy-listing-optimizer/main.py --listing data/listings/my-listing.json`
- `python3 skills/etsy-listing-optimizer/main.py --interactive`

**Inputs:**
- JSON file with `title`, `tags` (array), and `description` — see `data/listings/sample-listing.json` for format
- Preprocessed keyword data from `data/everbee-etsy/*.processed.md` (falls back to raw CSV)

**Outputs:** `reports/listing-optimizations/YYYY-MM-DD-<slug>.md`

**Output format:** Tags to remove (with reason) → replacement tags (with char count + score) → rewritten description opening → new title with keyword rationale

### `skills/creative-designer/`
**What it does:** Generates on-brand Pinterest pins end-to-end. Takes a list of topics, generates SEO copy via Claude, generates a background photo via Gemini Imagen (Nano Banana Pro), composites brand typography and overlay via Pillow, opens an HTML preview for review, then submits approved pins to Tailwind as drafts.

**Run:**
- `python3 skills/creative-designer/main.py` — interactive topic entry
- `python3 skills/creative-designer/main.py --topics "topic1" "topic2"` — inline topics
- `python3 skills/creative-designer/main.py --topics-file topics.txt` — one topic per line
- `python3 skills/creative-designer/main.py --skip-images` — skip Gemini (placeholder bg, good for copy testing)
- `python3 skills/creative-designer/main.py --skip-tailwind` — write queue file only, skip API submission

**Inputs:**
- Topics/keywords (manual entry, inline args, or text file)
- Context files in `context/` (loaded automatically)

**Outputs:**
- `outputs/pins/review/YYYY-MM-DD-HHMMSS/` — all generated pins + HTML preview
- `outputs/pins/approved/YYYY-MM-DD-HHMMSS/` — approved pins + `tailwind_queue.json`

**Pin spec:** 1000x1500px portrait, warm editorial photo background, dark overlay, serif headline (Noto Serif Display Light), all-caps category label (Montserrat), switzertemplates.com watermark.

**API keys used:** `ANTHROPIC_API_KEY` (copy), `GOOGLE_API_KEY` (images), `TAILWIND_API_KEY` (scheduling), `TAILWIND_BOARD_ID` (optional).

**Tailwind note (Phase 1):** Tailwind requires a hosted image URL. The queue file contains all metadata ready to paste. Phase 2 will add automated image hosting for full API submission.

**Additional requirements:** `pip3 install google-genai Pillow` (on top of base requirements). Fonts (Noto Serif Display Light, Montserrat) download automatically on first run.

### `scheduler/`
**What it does:** Runs continuously in the background. Every 14 days, sends a Mac desktop notification asking for a fresh Everbee export. When a new CSV appears in `data/everbee-etsy/`, automatically runs `report-generator` and sends a notification when the report is ready.

**Run:** `python3 scheduler/main.py` (keep running in terminal or add to login items)

**State:** `scheduler/.state.json` — tracks last notification date, last report date, and known CSV files by MD5 hash

**Dependencies:** stdlib only (no extra packages needed). Uses `osascript` for Mac notifications.

## Dashboard Update Protocol

Every agent must update `dashboard_data.json` in the project root after every run.
Do not skip this step. Do not update any section other than the one assigned to your agent.

### Etsy Expert agent — update `etsy_agent` section

After every run, write the following fields into the `etsy_agent` object:
- `last_run` — today's date (YYYY-MM-DD)
- `next_run` — 14 days from today (YYYY-MM-DD)
- `snapshot` — object with `total_sales`, `revenue_30d`, `top_listing`, `conversion_rate`, `avg_order_value` from the latest report
- `priority_tasks` — array of strings, top 5–10 priority tasks from the report (in order)
- `keyword_opportunities` — array of objects, each with `keyword`, `volume`, `score`
- `dead_listings` — array of objects, each with `title` and `views`

Leave all other top-level keys (`blog_seo_agent`, `keyword_masterlist`, `last_updated`) exactly as they are.

### Blog SEO agent — update `blog_seo_agent` section

After every run, update the `blog_seo_agent` object:
- `last_run` — today's date (YYYY-MM-DD)
- `posts_written` — append any newly written posts with: `keyword`, `date_written`, `word_count`, `output_filename`, `written: true`, `published: false`. Do not remove existing entries.
- `pending_tasks` — update to reflect the next keywords still to be written (remove any just completed)

Leave all other top-level keys exactly as they are.

### Keyword masterlist — update `keyword_masterlist` array

If the CSV at `agents/blog-seo-agent/keywords/switzertemplates_keyword_masterlist.csv` has been modified since the last run, replace the `keyword_masterlist` array in `dashboard_data.json` with the updated data. Each entry must include: `keyword`, `monthly_search_vol`, `competition`, `trend_pct`, `content_pillar`, `maps_to_product`, `priority_tier`, `score`, `pinterest_clicks_proven`.

Leave all other top-level keys exactly as they are.
